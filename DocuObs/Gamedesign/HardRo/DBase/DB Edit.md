Добавление кастомного предмета на сервер rAthena требует действий с двух сторон:

1. **Серверная часть** — описание предмета в базах данных сервера, чтобы он работал.
    
2. **Клиентская часть** — файлы для отображения картинок, названий и описаний в игре.
    

**Важно:** Способы для клиентской части отличаются в зависимости от версии вашего клиента Ragnarok Online. В этой инструкции будут описаны оба основных способа.

---

### **ЧАСТЬ 1: ДОБАВЛЕНИЕ ПРЕДМЕТА НА СЕРВЕР (СЕРВЕРНАЯ ЧАСТЬ)**

Сначала нужно добавить запись о предмете в базу данных сервера. Есть два способа: через текстовый файл или через SQL.

#### **Способ 1: Через текстовый файл (Рекомендуется для начинающих)**

1. Откройте файл `db/item_db2.txt` в вашей папке с сервером.
    
2. Вставьте новую строку в формате:
    
    text
    
    ID,DBName,ScreenName,Type,Price,Sell,Weight,ATK,DEF,Range,Slot,Job,Class,Gender,Loc,wLV,eLV,Refineable,View,{Script},{OnEquip_Script},{OnUnequip_Script}
    
3. **Пример добавления простого меча:**
    
    text
    
    30000,My_Sword,MySword,4,1000,500,100,150,0,1,2,0xFFFFFFFF,1,2,2,3,50,1,2,{ bonus bStr,5; },{},{}
    
    _Разбор примера:_
    
    - `30000` — **ID** предмета. Рекомендуется использовать высокие номера (30000 и выше), чтобы не конфликтовать с официальными предметами.
        
    - `My_Sword` — **DBName**. Внутреннее имя для сервера. Используется в консольной команде `@item`.
        
    - `MySword` — **ScreenName**. Название, которое увидят игроки.
        
    - `4` — **Type**. Тип: (4 = Броня / Оружие).
        
    - `1000` — **Price**. Цена покупки у NPC.
        
    - `500` — **Sell**. Цена продажи NPC (по умолчанию).
        
    - `100` — **Weight**. Вес предмета, умноженный на 10 (т.е. `100` = `10.0` веса).
        
    - `150` — **ATK**. Атака оружия.
        
    - `0` — **DEF**. Защита (для оружия = 0).
        
    - `1` — **Range**. Дальность атаки.
        
    - `2` — **Slot**. Количество слотов для карт.
        
    - `0xFFFFFFFF` — **Job**. Какие классы могут использовать. `0xFFFFFFFF` = все классы.
        
    - `1` — **Class**. Тип персонажа: (1 = обычный, 2 = транс, 4 = бэби, 8 = 3-й класс и т.д.).
        
    - `2` — **Gender**. Пол: (2 = любой).
        
    - `2` — **Loc**. Место экипировки: (2 = правая рука / оружие).
        
    - `3` — **wLV**. Уровень оружия (1-4).
        
    - `50` — **eLV**. Минимальный уровень персонажа для использования.
        
    - `1` — **Refineable**. Можно ли улучшать: (1 = да).
        
    - `2` — **View**. Как предмет выглядит на персонаже. `2` = одноручный меч.
        
    - `{ bonus bStr,5; }` — **Script**. Бонусы, которые дает предмет. Здесь прибавляет +5 к Силе.
        
    - `{}` и `{}` — Скрипты при надевании и снятии (в данном случае пустые).
        

#### **Способ 2: Через SQL (для продвинутых)**

Если вы используете SQL базу данных (что более надежно), добавьте запись в таблицу `item_db`. Структура таблицы аналогична, но названия колонок другие (например, `name_english`, `name_japanese`). Убедитесь, что вы добавляете новые ID (`id`), которые не используются.

### **ЧАСТЬ 2: НАСТРОЙКА ОТОБРАЖЕНИЯ В ИГРЕ (КЛИЕНТСКАЯ ЧАСТЬ)**

Теперь нужно, чтобы игроки видели название, описание и картинку предмета.

#### **Для старых клиентов (до 2012 года)**

В папке `data/` вашего клиента или в GRF-файле найдите и отредактируйте следующие файлы:

1. **`idnum2itemdisplaynametable.txt`** (Название)
    
    - Формат: `ID#Название_Предмета#`
        
    - Пример: `30000#MySword#`
        
2. **`idnum2itemdesctable.txt`** (Описание)
    
    - Формат: `ID#Строка 1#Строка 2#...`
        
    - Пример:
        
        text
        
        30000#
        Очень острый меч.#
        Атака: 150#
        Вес: 10#
        #
        
3. **`idnum2itemresnametable.txt`** (Спрайты)
    
    - Формат: `ID#Имя_Спрайта#`
        
    - `Имя_Спрайта` — это название ваших `.spr` и `.act` файлов без расширения.
        
    - Пример: `30000#My_Sword#`
        
    - **Файлы спрайтов:**
        
        - Для иконки в инвентаре: `data\texture\유저인터페이스\item\My_Sword.bmp` (24x24)
            
        - Для иконки в окне описания: `data\texture\유저인터페이스\collection\My_Sword.bmp` (75x100)
            
        - Для выброшенного на пол предмета: `data\sprite\아이템\My_Sword.spr` и `.act`
            
        - Для оружия на персонаже (муж.): `data\sprite\악세사리\남\남_My_Sword.spr` и `.act`
            
        - Для оружия на персонаже (жен.): `data\sprite\악세사리\여\여_My_Sword.spr` и `.act`
            
4. **`itemslotcounttable.txt`** (Слоты)
    
    - Добавьте предмет, только если у него больше 0 слотов.
        
    - Формат: `ID#Кол-во_слотов#`
        
    - Пример: `30000#2#`

#### **Для новых клиентов (после 2012 года)**

Всё управление перенесено в один файл: `System/ItemInfo.lub`.

1. **Сделайте резервную копию файла** `ItemInfo.lub`.
    
2. Сконвертируйте файл из `.lub` в `.lua` (или откройте его специальным редактором), чтобы редактировать его как текстовый.
    
3. Добавьте запись для вашего предмета в основной таблице:
    
    lua
    
    [30000] = {
        unidentifiedDisplayName = "Неизвестный меч",
        unidentifiedResourceName = "¼Òµå", -- Спрайт для неопознанного предмета
        unidentifiedDescriptionName = {
            "Неопознанный предмет. Можно опознать с помощью [Лупы]."
        },
        identifiedDisplayName = "MySword",
        identifiedResourceName = "My_Sword", -- Спрайт для опознанного предмета
        identifiedDescriptionName = {
            "Очень острый меч.",
            "Класс: ^777777Одноручный меч^000000",
            "Атака :^777777 150^000000",
            "Вес :^777777 10^000000",
            "Уровень оружия :^777777 3^000000",
            "Требуемый уровень :^777777 50^000000",
            "Применимые классы :^777777 Все классы^000000",
            "Бонус: Сила +5"
        },
        slotCount = 2,
        ClassNum = 2 -- Номер отображения на персонаже (2 = одноручный меч)
    },
    
4. После редактирования снова скомпилируйте `.lua` файл обратно в `.lub` с помощью компилятора Lua.

### **КАК ДОБАВИТЬ УНИКАЛЬНЫЙ ВНЕШНИЙ ВИД ДЛЯ ОРУЖИЯ (НОВЫЕ КЛИЕНТЫ)**

Если вы хотите, чтобы ваше оружие выглядело не как стандартный меч, а как уникальное, нужно добавить его в файл `weapontable.lub`.

1. Отредактируйте `data/luafiles514/lua files/datainfo/weapontable.lub`.
    
2. Добавьте новый тип оружия в таблицу `Weapon_IDs`:
    
    lua
    
    WEAPONTYPE_My_Sword = 98,
    
3. Свяжите его со спрайтом в таблице `WeaponNameTable`:
    
    lua
    
    [Weapon_IDs.WEAPONTYPE_My_Sword] = "_My_Sword"
    
4. Укажите, к какому классу оружия он относится, в таблице `Expansion_Weapon_IDs`:
    
    lua
    
    [Weapon_IDs.WEAPONTYPE_My_Sword] = Weapon_IDs.WPCLASS_WEAPONTYPE_SWORD
    
5. В файле `ItemInfo.lub` для вашего предмета укажите `ClassNum = 98` (используйте новый ID из `Weapon_IDs`).

### **КАК ОГРАНИЧИТЬ ВОЗМОЖНОСТЬ ПЕРЕДАЧИ ПРЕДМЕТА**

Чтобы сделать предмет персонализированным (нельзя выбросить, продать или передать), отредактируйте файл `db/item_trade.txt`.

- Формат: `ID, Маска_Запретов, Уровень_ГМа_для_обхода`
    
- **Маска_Запретов (суммируйте значения):**
    
    - `1` — Нельзя выбросить.
        
    - `2` — Нельзя торговать (или ставить в лавку).
        
    - `8` — Нельзя продать NPC.
        
    - `32` — Нельзя положить в склад.
        
- **Пример:** Сделать предмет `30000` нельзя выбрасывать и торговать:  
    `30000,3,99` (3 = 1+2, `99` = только ГМ 99 уровня может игнорировать правило).

### **ГДЕ ВЗЯТЬ СПРАЙТЫ ДЛЯ НОВОГО ПРЕДМЕТА?**

1. Скачайте готовый набор файлов для предмета (например, с форумов rAthena).
    
2. В наборе обычно есть 6 файлов:
    
    - `drop.spr` / `drop.act` (на полу).
        
    - `item.bmp` (иконка 24x24).
        
    - `collection.bmp` (картинка 75x100).
        
    - `spr` / `act` (вид на персонаже).
        
3. Переименуйте их в соответствии с `identifiedResourceName` (в вашем случае `My_Sword.spr` и т.д.) и разложите по папкам, указанным в инструкции.
